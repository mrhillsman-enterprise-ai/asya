# Compiled Flows Examples

This directory contains examples of compiled flows generated by the `asya flow compile` command. Each subdirectory shows the output for different flow patterns.

## What Are Compiled Flows?

Compiled flows are auto-generated router actors that implement control-flow logic (conditionals, mutations, routing) for your data pipelines. The Flow DSL compiler transforms Python flow definitions into these routers.

**Flow Structure**:
- **Source Flow** (`examples/flows/*.py`) - Python function describing your pipeline
- **Compiled Routers** (`compiled/*/routers.py`) - Generated control-flow actors
- **Handler Actors** (your code) - ML models, data processors (deployed separately)

## Router Naming Convention

Generated routers follow a consistent naming pattern:

| Pattern | Example | Description |
|---------|---------|-------------|
| `start_{flowname}` | `start_sample_flow` | Flow entrypoint |
| `router_{flowname}_line_{N}_if` | `router_sample_flow_line_3_if` | Conditional branching at line N |
| `router_{flowname}_line_{N}_seq` | `router_sample_flow_line_4_seq` | Sequential mutations at line N |
| `end_{flowname}` | `end_sample_flow` | Flow exitpoint |

**Line Numbers**: Line numbers in router names match the source flow file for easy debugging.

```python
def if_else_simple_flow(p: dict) -> dict:
```

### Mutations in Branches

**Source** (`../if_mutations_in_branches.py`):
```python
def if_mutations_in_branches_flow(p: dict) -> dict:
    p = handler_setup(p)
    if p["type"] == "A":
        p["branch"] = "A"  # Inline mutation
        p = handler_type_a(p)
    else:
        p = handler_type_b(p)
        p["branch"] = "B"  # Inline mutation
    p = handler_finalize(p)
    return p
```

**Generated Routers**:
- `start_if_mutations_in_branches_flow` - Routes to `handler_setup`
- `router_if_mutations_in_branches_flow_line_3_if` - Conditional branching
- `router_if_mutations_in_branches_flow_line_4_seq` - Mutations for "A" branch
- `router_if_mutations_in_branches_flow_line_8_seq` - Mutations for "B" branch
- `end_if_mutations_in_branches_flow` - Exitpoint

**Key Insight**: Payload mutations (`p["branch"] = "A"`) are compiled into separate routers that execute BEFORE the next handler call.

**Use Case**: Branch-specific metadata, tracking decisions.

---

**Use Case**: Input validation, error handling, conditional termination.

## Handler Resolution

All generated routers use the `resolve()` function to map handler names to actor names at runtime.

### How It Works

1. **Flow Compilation**: Handlers referenced in source flow (`handler_setup`, `handler_type_a`)
2. **Runtime Resolution**: `resolve("handler_setup")` looks up environment variables
3. **Suffix Matching**: Supports flexible matching of handler paths

### Environment Variable Format

```bash
ASYA_HANDLER_<ACTOR_NAME>="full.module.path.ClassName.method"
```

**Example**:
```bash
# Handler actor deployments
ASYA_HANDLER_SETUP="handlers.setup.process"
ASYA_HANDLER_TYPE_A="handlers.type_a.TypeAHandler.process"
ASYA_HANDLER_TYPE_B="handlers.type_b.TypeBHandler.process"

# In router deployment, set same mappings:
ASYA_HANDLER_SETUP="handlers.setup.process"
ASYA_HANDLER_TYPE_A="handlers.type_a.TypeAHandler.process"
ASYA_HANDLER_TYPE_B="handlers.type_b.TypeBHandler.process"
```

### Suffix Matching

```python
# All of these work (if unambiguous):
resolve("process")                              # Shortest suffix
resolve("TypeAHandler.process")                 # Class + method
resolve("type_a.TypeAHandler.process")          # Module + class + method
resolve("handlers.type_a.TypeAHandler.process") # Full path
```

**Ambiguity Error**:
```bash
# If two handlers end with "process":
ASYA_HANDLER_A="module1.process"
ASYA_HANDLER_B="module2.process"

# Then resolve("process") raises ValueError
# Use longer suffix: resolve("module1.process")
```

## Deployment Example

Here's how to deploy the `if_else_simple` flow:

### 1. Build Router Image

```dockerfile
FROM python:3.13-slim
WORKDIR /app
COPY if_else_simple/routers.py /app/routers.py
CMD ["python3", "-c", "import routers; print('Routers loaded')"]
```

```bash
docker build -t if-else-routers:v1 .
```

### 2. Deploy Router Actors

**Start Router**:
```yaml
apiVersion: asya.sh/v1alpha1
kind: AsyncActor
metadata:
  name: start-if-else-simple-flow
spec:
  transport: sqs
  workload:
    kind: Deployment
    template:
      spec:
        containers:
        - name: asya-runtime
          image: if-else-routers:v1
          env:
          - name: ASYA_HANDLER
            value: "routers.start_if_else_simple_flow"
          # Handler mappings
          - name: ASYA_HANDLER_SETUP
            value: "handlers.setup.process"
          - name: ASYA_HANDLER_TYPE_A
            value: "handlers.type_a.process"
          - name: ASYA_HANDLER_TYPE_B
            value: "handlers.type_b.process"
          - name: ASYA_HANDLER_FINALIZE
            value: "handlers.finalize.process"
```

**Conditional Router**:
```yaml
apiVersion: asya.sh/v1alpha1
kind: AsyncActor
metadata:
  name: router-if-else-simple-flow-line-10-if
spec:
  transport: sqs
  workload:
    kind: Deployment
    template:
      spec:
        containers:
        - name: asya-runtime
          image: if-else-routers:v1
          env:
          - name: ASYA_HANDLER
            value: "routers.router_if_else_simple_flow_line_10_if"
          # Same handler mappings as start router
          - name: ASYA_HANDLER_SETUP
            value: "handlers.setup.process"
          - name: ASYA_HANDLER_TYPE_A
            value: "handlers.type_a.process"
          - name: ASYA_HANDLER_TYPE_B
            value: "handlers.type_b.process"
          - name: ASYA_HANDLER_FINALIZE
            value: "handlers.finalize.process"
```

**End Router**:
```yaml

  name: end-if-else-simple-flow

  transport: sqs

      spec:
        containers:
```

## Testing Compiled Flows

### Unit Test Generated Routers

Test router logic independently:

```python
# test_routers.py
import os
from if_else_simple.routers import router_if_else_simple_flow_line_10_if

# Set up handler mappings
os.environ['ASYA_HANDLER_TYPE_A'] = "handlers.type_a.process"
os.environ['ASYA_HANDLER_TYPE_B'] = "handlers.type_b.process"
os.environ['ASYA_HANDLER_FINALIZE'] = "handlers.finalize.process"

def test_router_type_a():
    message = {
        'payload': {'type': 'A'},
        'route': {'actors': [], 'current': 0}
    }
    result = router_if_else_simple_flow_line_10_if(message)

    # Verify routing to type_a handler
    assert 'handler-type-a' in result['route']['actors']
    assert 'handler-finalize' in result['route']['actors']

def test_router_type_b():
    message = {
        'payload': {'type': 'B'},
        'route': {'actors': [], 'current': 0}
    }
    result = router_if_else_simple_flow_line_10_if(message)

    # Verify routing to type_b handler
    assert 'handler-type-b' in result['route']['actors']
    assert 'handler-finalize' in result['route']['actors']
```

## Common Patterns

### Pattern 1: Validation Flow

```python
def validation_flow(p: dict) -> dict:
    p = validate_schema(p)
    if not p.get("valid"):
        return p  # Early exit
    p = validate_business_rules(p)
    if not p.get("valid"):
        return p
    p = process_valid_data(p)
    return p
```

**Generated**: Multiple validation routers with early exit paths.

### Pattern 2: Multi-Model Routing

```python
def multi_model_flow(p: dict) -> dict:
    if p["language"] == "en":
        p = english_model(p)
    elif p["language"] == "es":
        p = spanish_model(p)
    elif p["language"] == "fr":
        p = french_model(p)
    else:
        p["error"] = "Unsupported language"
        return p
    p = postprocess(p)
    return p
```

**Generated**: Multi-branch router for language-specific models.

### Pattern 3: Progressive Enhancement

```python
def progressive_enhancement_flow(p: dict) -> dict:
    p = base_features(p)
    if p.get("premium_user"):
        p = premium_features(p)
    if p.get("experimental_enabled"):
        p = experimental_features(p)
    return p
```

**Generated**: Sequential routers with optional enhancement branches.

## Best Practices

### Router Deployment

1. **Single image for all routers**: Package all routers in one image, deploy with different `ASYA_HANDLER` values
2. **Consistent handler mappings**: Use same `ASYA_HANDLER_*` environment variables across all router deployments
3. **Minimal resources**: Routers are lightweight (no ML models), use small resource limits
4. **No autoscaling needed**: Routers execute quickly, scale 1:1 with handler actors

### Handler Mappings

1. **Use descriptive actor names**: `handler-setup`, `handler-type-a` (kebab-case, lowercase)
2. **Avoid ambiguous suffixes**: If `resolve("process")` is ambiguous, use longer suffix like `TypeA.process`
3. **Document mappings**: Maintain a mapping reference file for platform teams

### Debugging

1. **Check line numbers**: Router names reference source flow line numbers
2. **Visualize flows**: Use `--plot` to generate flow diagrams
3. **Test routers independently**: Unit test router logic before deployment
4. **Trace message flow**: Use message IDs to track routing through routers

## Limitations

Current Flow DSL supports basic control flow only:

**Supported**:
- Conditionals (`if`/`elif`/`else`)
- Payload mutations (`p["key"] = value`)
- Sequential handler calls (`p = handler(p)`)
- Class instantiation (default args only)
- Early returns

**Not Supported**:
- Loops (`for`, `while`)
- Dynamic routing based on runtime conditions
- Multiple return values (`p, q = handler(p)`)
- Non-default constructor arguments
